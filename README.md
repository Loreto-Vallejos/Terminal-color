### 1. ¿Qué aprendiste sobre dependencias y npm?

Aprendimos que npm es el gestor de paquetes de **Node.js** que nos permite instalar dependencias externas (bibliotecas) para nuestro proyecto, añadiendo nuevas funcionalidades sin necesidad de desarrollarlas desde cero.
En nuestro caso descargamos la libreria **`chalk`**, que nos permitio añadir color al texto mostrado en la consola, diferenciando mejor los mensajes del tipo advertencia, error o información.
Ademas el uso de npm nos permitio gestionar esta dependencia de forma eficiente, iniciando el proyecto con el comando **`npm init -y`** y posteriormente ejecutar la descarga de la dependencia chalk mediante **`npm install chalk`**. 



### 2. ¿Por qué es importante separar trabajo en ramas?

Separar el trabajo en ramas nos permite crear un entorno de desarrollo seguro para la rama principal, al ejecutar nuestras tareas en una rama diferente. 
De esta manera, cada miembro del equipo puede trabajar en una funcionalidad o modificacion sin afectar el trabajo de los demás, lo que permite un trabajo colaborativo, ordenado y seguro.


### 3. ¿Qué diferencia hay entre `console.log`, `console.info`, `console.warn` y `console.error`?, ¿Qué errores comunes encontraron en equipo?

Las funciones **`console`** en JavaScript se utilizan para depurar código y mostrar información en la consola. 

- **`console.log`**: Muestra el texto en la consola como un mensaje general.
- **`console.info`**:Muestra el texto en la consola como un mensaje informativo.
- **`console.warn`**: Muestra el texto en la consola como advertencia.
- **`console.error`**: Muestra el texto en la consola como un mensaje de error o falla.

Como equipo, algunos de los errores mas comunes fueron principalmente de sintaxis al ingresar comandos en la terminal.
